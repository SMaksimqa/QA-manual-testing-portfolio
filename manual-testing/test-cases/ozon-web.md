# Test Case #2: Фильтр по цене в поиске на Ozon (positive + negative)

ID: TC_OZON_SEARCH_001  
Название: Проверка работы фильтра по цене в результатах поиска  
Предусловие: Открыт браузер, пользователь не авторизован, перейдена главная страница https://www.ozon.ru  
Окружение: Chrome latest / Windows 11 / мобильная версия (responsive)  
Приоритет: High  
Тип теста: Functional / UI / E2E  

### Positive сценарий

Шаги:
1. В поисковую строку ввести запрос: ноутбук
2. Нажать Enter / кнопку поиска
3. В блоке фильтров слева нажать на раздел "Цена"
4. Указать диапазон: от 50 000 до 100 000 ₽
5. Применить фильтр (кнопка "Применить" или автоматическое применение)

Ожидаемый результат:
- В результатах поиска отображаются товары только в диапазоне 50 000 – 100 000 ₽ (± небольшая погрешность 1–3%)
- Цены на карточках товаров соответствуют фильтру
- Фильтр отображается как активный вверху страницы
- Нет товаров дешевле 50к или дороже 100к

Постусловие: Сбросить фильтры

### Negative сценарии

Негатив 1: Минимальная цена выше максимальной  
Шаги: От 120 000 до 80 000 ₽ → Применить  
Ожидаемый результат: Ошибка «Минимальная цена не может быть больше максимальной» или фильтр не применяется

Негатив 2: Пустые поля в фильтре  
Шаги: Оставить "от" и "до" пустыми → Применить  
Ожидаемый результат: Фильтр игнорируется / показываются все товары

Негатив 3: Ввод букв вместо цифр  
Шаги: В поле "от" ввести "abc" → Применить  
Ожидаемый результат: Поле не принимает буквы / появляется ошибка / значение не применяется

Негатив 4: Ввод очень большой суммы (например 1 000 000 000 ₽)  
Ожидаемый результат: Фильтр применяется, но результатов может не быть (или показывается "Ничего не найдено")

Негатив 5: Фильтр после смены категории  
Шаги: Применить фильтр 50–100к → перейти в другую категорию (например "Смартфоны")  
Ожидаемый результат: Фильтр сбрасывается / сохраняется (зависит от поведения сайта)
