# (Ru) QA practice — Booking.com (mobile web)

**Продукт**: [Booking.com](http://booking.com/)

**Платформа**: Mobile Web

**Устройство**: iPhone

**Браузер**: Chrome

**Тип тестирования**: Exploratory testing (исследовательское)

**Параметры поиска:**
• Локация: Haad Rin
• Даты: 10–12 апреля
• Количество взрослых: 8–15
• Количество ночей: 2

**Цель:**
Проверить, как отображаются рекомендованные варианты размещения
для групп разного размера.

**Тип:** UI / UX

**Severity:** Medium

**Окружение:**
• Платформа: Mobile Web
• Браузер: Chrome

**Шаги воспроизведения:**

1. Открыть страницу отеля Sun Beach Bungalows Resort
2. Установить параметры поиска для 8–10 взрослых
3. Прокрутить страницу до блока «Рекомендовано для группы»

**Фактический результат:**

Цена отображается не полностью.
Часть стоимости обрезана, горизонтальная прокрутка отсутствует,
увидеть цену целиком невозможно.

**Ожидаемый результат:**

Цена должна отображаться полностью
или быть доступной через прокрутку.

**Детали воспроизведения:**

- Баг воспроизводится при выборе 8–10 взрослых
- Баг НЕ воспроизводится при выборе 15 взрослых
- Тип апартаментов меняется в зависимости от количества гостей
- Отображение блока «Рекомендовано для группы» ведёт себя нестабильно
- Поведение зависит от конфигурации данных

**Примечания:**

- Обычные (не рекомендованные) варианты отображаются корректно-
-  Проблема наблюдается не во всех отелях

**Скриншоты**

- Скриншот с некорректным отображением (цена обрезана)
- Скриншот с корректным отображением рекомендованного варианта

**Итог сессии**

- Найден UI/UX баг в динамическом блоке «Рекомендовано для группы»
- Поведение блока зависит от количества гостей и типа апартаментов
- В ряде случаев скрывается критичная информация — цена
- Условия воспроизведения успешно выявлены и подтверждены
