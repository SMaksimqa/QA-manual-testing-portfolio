# Qa practice - Ozon (web)

# 🔍 Тестирование поиска — спецсимволы

**Сайт**: Ozon (web-версия)

**Раздел**: Поиск товаров

**Дата**: 28.01.2026

**Тип тестирования**: Функциональное / Валидация пользовательского ввода

🎯 **Цель проверки**

Проверить, как поиск на сайте Ozon обрабатывает запросы, состоящие из специальных символов.

**Используемые спецсимволы**:

- / : ; ( ) ₽ & @ " . , ? !

# 🧪 Результаты тестирования

❌ **Кейс 1**: Один спецсимвол

Пример ввода: !

**Фактический результат**:

- Отображается подсказка (validation message)
- Поиск не выполняется
- Результаты не отображаются

**Ожидаемый результат**:

- Единообразная обработка невалидного ввода
- Либо блокировка поиска с поясняющим сообщением
- Либо пустая выдача без выполнения поиска

**Статус**: 🐞 Баг

❌**Кейс 2**: Два спецсимвола

**Пример ввода**: !!

**Фактический результат**:

- Отображается та же подсказка, что и при одном спецсимволе
- Поиск выполняется
- Отображается нерелевантная (рандомная) выдача товаров

**Ожидаемый результат**:

- **Консистентное поведение**:
    - либо поиск блокируется, как в кейсе с одним спецсимволом;
    - либо отображается сообщение «Ничего не найдено» без нерелевантных данных.
- 

**Статус**: 🐞 Баг

# ⚠️ Выявленные проблемы

1. Нарушена консистентность валидации поискового ввода:  
2. Oдинаково невалидные данные (! и !!) обрабатываются по-разному.
3. Отсутствует корректная обработка запросов без смысловой нагрузки.
4. Пользователь получает вводящую в заблуждение выдачу товаров.

# 🐞 Предполагаемый дефект

Тип: Functional / UX Bug

Описание: Поиск на Ozon (web) некорректно обрабатывает запросы, состоящие из специальных символов, что приводит к несогласованному поведению и нерелевантной выдаче.

# 💡 Комментарий QA

Поведение поиска при невалидном вводе должно быть единообразным. Текущее поведение может вводить пользователя в заблуждение и снижать доверие к поисковой системе.
